GitHub Repo Technical Whitepaper

NanoFusion is a protocol and implementation for trustless mixing of funds on the Nano cryptocurrency network. Trustless mixing improves financial privacy for Nano users. NanoFusion achieves trustless mixing by using a binary tree of aggregate-signature accounts to combine funds from many accounts into one (without any risk that funds could be stolen), then sending those funds to new accounts controlled by the original owners.

Some cool technical details:

  • NanoFusion is loosely inspired by CashFusion for Bitcoin Cash.
  • NanoFusion implements the Ed25519 Aggregated Signature algorithm published by KZenNetworks, but using Blake2B hashes for Nano compatibility.
  • NanoFusion works purely in client-side Javascript, requiring zero changes to the existing Nano node software.
